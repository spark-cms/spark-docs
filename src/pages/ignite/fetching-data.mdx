# Fetching Data

Ignite lets you fetch multiple types of data in different ways.

## `fetchContent`
The `fetchContent` function allows you to fetch a specific type of content from a page.

### Parameters
The following parameters are required:

| Parameter | Type     | Description                              |
|-----------|----------|------------------------------------------|
| `page`    | `string` | The API address of the page.             |
| `content` | `string` | The API address of the content to fetch. |

The following parameters are optional:
| Parameter | Type       | Description                                                     |
|-----------|------------|-----------------------------------------------------------------|
| `fields`  | `string[]` | The fields which to include. Leave empty to include everything. |

### Example
```js filename="ignite.js" showLineNumbers copy
// Import the Ignite class
import { Ignite } from "@spark-cms/ignite";

// Create a new Ignite instance
const spark = new Ignite({
  name: "John Doe",
  project: "Private Blog",
  token: "SPARK-a216ee62-b106-447d-98ff-e99cd70d3340",
});

// Fetch content from the CMS.
const content = await spark.fetchContent("blog", "posts");
```

The `content` variable will now contain an array of all the posts on the blog.

## `fetchContentByIdentifier`
The `fetchContentByIdentifier` function allows you to fetch a specific type of content from a page by it's identifier.

<Callout>
  **Good to know:** 
  <br />The identifier can either be an ID or a slug. To learn more about slugs, check out the [Slugs](/docs/slugs) page. {/* TODO */}
</Callout>

### Parameters
The following parameters are required:

| Parameter | Type     | Description                                                       |
|-----------|----------|-------------------------------------------------------------------|
| `page`    | `string` | The API address of the page.                                      |
| `content` | `string` | The identifier of the content to fetch. Format: `type:identifier` |

The following parameters are optional:

| Parameter | Type       | Description                                                     |
|-----------|------------|-----------------------------------------------------------------|
| `fields`  | `string[]` | The fields which to include. Leave empty to include everything. |

### Example
```js filename="ignite.js" showLineNumbers copy
// Import the Ignite class
import { Ignite } from "@spark-cms/ignite";

// Create a new Ignite instance
const spark = new Ignite({
  name: "John Doe",
  project: "Private Blog",
  token: "SPARK-a216ee62-b106-447d-98ff-e99cd70d3340",
});

// Fetch content from the CMS.
const content = await spark.fetchContentByIdentifier("blog", "posts:aezg4x1g3zdsn7afusyy");

// Using a slug
const content = await spark.fetchContentByIdentifier("blog", "posts:my-first-post");
```

The `content` variable will now contain the first post on the blog.

## `fechPage`
The `fetchPage` function allows you to fetch an entire page from the CMS.

### Parameters
The following parameters are required:

| Parameter | Type     | Description                  |
|-----------|----------|------------------------------|
| `page`    | `string` | The API address of the page. |

The following parameters are optional:

| Parameter | Type       | Description                                                                                   |
|---------- |------------|---------------------------------------------------------------------------------------------- |
| `content` | `string[]` | The API addresses of the content which should be included. Leave empty to include everything. |

### Example
```js filename="ignite.js" showLineNumbers copy
// Import the Ignite class
import { Ignite } from "@spark-cms/ignite";

// Create a new Ignite instance
const spark = new Ignite({
  name: "John Doe",
  project: "Private Blog",
  token: "SPARK-a216ee62-b106-447d-98ff-e99cd70d3340",
});

// Fetch a page from the CMS.
const page = await spark.fetchPage("blog");
```

The `page` variable will now contain the entire blog page, all posts and all other content.

### Parameters
The following parameters are required:

| Parameter | Type       | Description                                                     |
|-----------|------------|-----------------------------------------------------------------|

| `fields`  | `string[]` | The fields which to include. Leave empty to include everything. |

### Example
```js filename="ignite.js" showLineNumbers copy
// Import the Ignite class
import { Ignite } from "@spark-cms/ignite";

// Create a new Ignite instance
const spark = new Ignite({
  name: "John Doe",
  project: "Private Blog",
```
